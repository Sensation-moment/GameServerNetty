一个基于Netty的简易游戏服务器框架

- 项目框架
  - JDK 11
  - Maven 3.8.2
  - SpringBoot 2
  - Netty 4.1.39
  - MyBatis 3.5.3
  - MySQL 5.7
  - Redis
  - RocketMQ
  - Google Protobuf
  
- 实现的功能

  - 使用Google ProtoBuf定义消息实体，并转化为Java类

  - 客户端和服务端建立连接，并且能够互通消息(通过Netty进行收发消息)
  - 消息编解码处理(Netty定义解码器、编码器、中间处理器)
  - 消息协议分发机制(不同的消息种类识别，并映射发送到不同的处理模块)
  - 线程模型
    - 线程池的搭建：任务逻辑的线程模型
  - 基本游戏逻辑
    - 创建账号
    - 登录账号
    - 英雄角色的创建
    - 英雄角色加入属性(攻击力和血量)
    - 加入战斗的逻辑，实现多个角色的互殴
    - 防止玩家互殴导致线程并发问题，战斗逻辑采用单线程处理
    - 使用Redis缓存部分数据
    - 将击杀信息写入消息队列，实现排行榜
